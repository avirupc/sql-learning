# `GROUP BY` and `HAVING`

(Sources for following part: https://sqlzoo.net/wiki/SUM_and_COUNT and https://sqlzoo.net/wiki/Using_GROUP_BY_and_HAVING.)

By including a `GROUP BY` clause functions such as `SUM` and `COUNT` are applied to groups of items sharing values. When we specify `GROUP BY continent` the result is that we get only one row for each different value of continent. All the other columns must be "aggregated" by one of `SUM`, `COUNT` ...

The `HAVING` clause allows us to filter the groups which are displayed. The `WHERE` clause filters rows before the aggregation, the `HAVING` clause filters after the aggregation.

If an `ORDER BY` clause is included we can refer to columns by their position. 

### Examples:

#### For each continent show the number of countries

```SQL
SELECT continent, COUNT(name) FROM world
GROUP BY continent
```

#### For each continent show the total population

```SQL
SELECT continent, SUM(population) FROM world
GROUP BY continent
```

## `WHERE` and `GROUP BY`

The WHERE filter takes place **before** the aggregating function. 

### Example:

#### For each relevant continent show the number of countries that has a population of at least 200000000. 

```SQL
SELECT continent, COUNT(name)
  FROM world
 WHERE population >= 200000000
 GROUP BY continent
```

## `GROUP BY` and `HAVING`

The `HAVING` clause is tested **after** the `GROUP BY`. You can test the aggregated values with a `HAVING` clause. 

### Example:

#### Show the total population of those continents with a total population of at least half a billion

```SQL
SELECT continent, SUM(population)
FROM world
GROUP BY continent
HAVING SUM(population) >= 500000000
```

### 1, List the continents that have a total population of at least 100 million. 

<details>
<summary>🟩Solution</summary>

```SQL
SELECT continent from world
GROUP BY continent
HAVING SUM(population) >= 100000000
```
</details>

---

### 2. For each subject show the subject and the number of prizes. 

<details>
<summary>🟩Solution</summary>

```SQL
SELECT subject, COUNT(winner) 
FROM nobel
GROUP BY subject
```
</details>

### 3. For each subject show the first year that the prize was awarded. 

<details>
<summary>🟩Solution</summary>

```SQL
SELECT subject, MIN(yr)
FROM nobel
GROUP BY subject
```
</details>

### 4. For each subject show the number of prizes awarded in the year 2000.

<details>
<summary>🟩Solution</summary>

```SQL
SELECT subject, COUNT(winner)
FROM nobel
WHERE yr = 2000
GROUP BY subject
```
</details>

### 5. Show the number of different winners for each subject. Be aware that Frederick Sanger has won the chemistry prize twice - he should only be counted once. 

<details>
<summary>🟩Solution</summary>

```SQL
SELECT subject, COUNT(DISTINCT(winner))
FROM nobel
GROUP BY subject
```
</details>

### 6. For each subject show how many years have had prizes awarded. 

<details>
<summary>🟩Solution</summary>

```SQL
SELECT subject, COUNT(DISTINCT(yr))
FROM nobel
GROUP BY subject
```
</details>

### 7. Show the years in which three prizes were given for Physics. 

<details>
<summary>🟩Solution</summary>

```SQL
SELECT yr
FROM nobel
WHERE subject = 'Physics'
GROUP BY yr
HAVING COUNT(winner) = 3 
```
</details>

### 8. Show winners who have won more than once. 

<details>
<summary>🟩Solution</summary>

```SQL
SELECT winner FROM nobel
GROUP BY winner
HAVING count(*) > 1
```
</details>
<br>

⚠️**Note:** <br>
When counting rows in SQL, `COUNT(*)` is generally preferred over `COUNT(column)` for correctness and reliability.

✅ Key Difference:
- `COUNT(*)` → counts all rows, regardless of NULL values
- `COUNT(column)` → counts only rows where the column is NOT NULL

### 9. Show winners who have won more than one subject. 

<details>
<summary>🟩Solution</summary>

```SQL
SELECT winner 
FROM nobel
GROUP BY winner
HAVING COUNT(DISTINCT(subject)) > 1
```
</details>

### 10. Show the year and subject where 3 prizes were given. Show only years 2000 onwards. 

<details>
<summary>🟩Solution</summary>

```SQL
SELECT yr, subject FROM nobel
WHERE yr >= 2000
GROUP BY yr, subject
HAVING COUNT(*) = 3
```
</details>
